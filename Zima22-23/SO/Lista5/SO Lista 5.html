Zadanie 1

To, że rura jest jednokierunkowa, oznacza, że komunikacja między procesami działa tylko w jedną stronę, czyli wyjście jednego procesu staje się wejściem drugiego procesu (nie może być jednocześnie komunikacji w obie strony).

  • Dla pustej/zapełnionej rury, odpowiednio read/write zostają zablokowane.

  • Dodatkowo jeżeli rura jest pusta oraz zamknięty został koniec do zapisu, to dostajemy shortcount’a.

  • W przypadku, gdy więcej kilka (więcej niż jeden) procesów współbieżnie zapisuje do rury wiersze tekstu nie większe niż PIPE_BUF, mamy gwarancję, że nie zostaną one pofragmentowane, tj. jeżeli procesy A, B piszą do rury, to w rurze znajdą się nie podzielone bloki [A][B] lub [B][A]

    • Przykład gdy jest to przydatne: gdy wiele procesów wyrzuca za pomocą jednego potoku pewne logi na dysk i nie są one większe niż PIPE_BUF, to mamy gwarancję, że jądro nie złamie nam tych linii, tylko będzie je zapisywał w całości.

  • Gdy w poleceniu ps -ef | grep sh | wc -l, wc -l zakończy swoje działanie, nastąpi zamknięcie jego końca do odczytu, ponieważ nie ma już procesu który będzie te dane czytał. Wtedy poprzedni proces, czyli grep sh, przy próbie zapisu do rury otrzyma SIGPIPE (którego domyślną akcją jest zakończenie działania procesu), ponieważ jądro będzie wiedziało, że nie istnieje proces, który przeczyta informacje, które będzie probował zapisać do rury grep. SIGPIPE propaguje się na pozostałe procesy, co sprawia że zostają one wszystkie zakończone. (Jeśli zabijemy którykolwiek z procesów z potoku pozostałe programy też dostaną SIGPIPE.)

Short count może występować w następujących sytuacjach:

  • gdy podczas odczytu natrafiamy na koniec pliku (EOF)
  • gdy czytamy wiersz z terminala, ale okazał się on krótszy niż nasz bufor
  • gdy czytamy/zapisujemy do gniazd sieciowych
  • When writing up to PIPE_BUF bytes, write() will block if necessary until sufficient space is available in the pipe so that it can complete the operation atomically. When more than PIPE_BUF bytes are being written, write() transfers as much data as possible to fill the pipe, and then blocks until data has been removed from the pipe by some reading process. If such a blocked write() is interrupted by a signal handler, then

the call unblocks and returns a count of the number of bytes successfully trans-
ferred, which will be less than was requested (a so-called partial write).

Short count nie występuje w przypadku plików zwykłych przechowywanych na dysku (z wyjątkiem EOF podczas odczytywania)

  • Rury przy odczycie zachowują się jak zwykłe pliki, tj. short count zostaje zwrócony tylko gdy nie ma danych do odczytu oraz wiemy, że już ich więcej nie będzie.

  • Operacja write może zwrócić short counta w sytuacji, gdy np. na dysku zabraknie pamięci

  • Jak można połączyć rodzica i dziecko rurą, która została utworzona po uruchomieniu dziecka?

Zadanie 2

ioctl and fcntl Are an Embarassment

Over the years, ioctl and fcntl calls have proliferated. They are often poorly documented, and often a source of portability problems as well. With each one comes a grubby pile of macro definitions describing operation types and special argument values.


Wywołanie systemowe ioctl() (input/output control) służy do specyficznych dla urządzenia operacji input/output oraz innych operacji, które nie mogą zostać wykonane za pomocą zwykłych wywołań systemowych.

The kernel generally dispatches an ioctl call straight to the device driver, which can interpret the request number and data in whatever way required. The writers of each driver document request numbers for that particular driver and provide them as constants in a header file.

int ioctl(int fd, unsigned long request, ...);
  • fd musi być otwartym deskryptorem pliku
  • request: The second argument is a device-dependent request code. An ioctl() request has encoded in it whether the argument is an in parameter or out parameter, and the size of the argument argp in bytes.
  • argp (trzeci argument): The third argument is an untyped pointer to memory. It’s traditionally char *argp (from the days before void * was valid C)
  • argp v2: an untyped pointer to data (either going to the driver, coming back from the driver, or both).

/*
* Ioctl's have the command encoded in the lower word, and the size of
* any in or out parameters in the upper word.  The high 3 bits of the
* upper word are used to encode the in/out status of the parameter.
*
*	 31 29 28                     16 15            8 7             0
*	+---------------------------------------------------------------+
*	| I/O | Parameter Length        | Command Group | Command       |
*	+---------------------------------------------------------------+
*/
#define	IOCPARM_MASK	0x1fff		/* parameter length, at most 13 bits */
#define	IOCPARM_SHIFT	16
#define	IOCGROUP_SHIFT	8
#define	IOCPARM_LEN(x)	(((x) >> IOCPARM_SHIFT) & IOCPARM_MASK)
#define	IOCBASECMD(x)	((x) & ~(IOCPARM_MASK << IOCPARM_SHIFT))
#define	IOCGROUP(x)	(((x) >> IOCGROUP_SHIFT) & 0xff)

#include "include/csapp.h"

#define DIRBUFSZ 256

static void print_mode(mode_t m) {
  char t;

  // katalog
  if (S_ISDIR(m))
    t = 'd';
  // znakowe
  else if (S_ISCHR(m))
    t = 'c';
  // blokowy
  else if (S_ISBLK(m))
    t = 'b';
  // zwykly
  else if (S_ISREG(m))
    t = '-';
  // fifo
  else if (S_ISFIFO(m))
    t = 'f';
  // link symb
  else if (S_ISLNK(m))
    t = 'l';
  // socket
  else if (S_ISSOCK(m))
    t = 's';
  // else
  else
    t = '?';

  char ur = (m & S_IRUSR) ? 'r' : '-';
  char uw = (m & S_IWUSR) ? 'w' : '-';
  char ux = (m & S_IXUSR) ? 'x' : '-';
  char gr = (m & S_IRGRP) ? 'r' : '-';
  char gw = (m & S_IWGRP) ? 'w' : '-';
  char gx = (m & S_IXGRP) ? 'x' : '-';
  char or = (m & S_IROTH) ? 'r' : '-';
  char ow = (m & S_IWOTH) ? 'w' : '-';
  char ox = (m & S_IXOTH) ? 'x' : '-';

  /* TODO: Fix code to report set-uid/set-gid/sticky bit as 'ls' does. */
  ux = (S_ISUID & m) ? ((m & S_IXUSR) ? 's' : 'S') : ux;
  gx = (S_ISGID & m) ? ((m & S_IXGRP) ? 's' : 'S') : gx;
  ox = (S_ISVTX & m) ? ((m & S_IXOTH) ? 't' : 'T') : ox;
  printf("%c%c%c%c%c%c%c%c%c%c", t, ur, uw, ux, gr, gw, gx, or, ow, ox);
}

static void print_uid(uid_t uid) {
  struct passwd *pw = getpwuid(uid);
  if (pw)
    printf(" %10s", pw->pw_name);
  else
    printf(" %10d", uid);
}

static void print_gid(gid_t gid) {
  struct group *gr = getgrgid(gid);
  if (gr)
    printf(" %10s", gr->gr_name);
  else
    printf(" %10d", gid);
}

static void file_info(int dirfd, const char *name) {
  struct stat sb[1];

  /* TODO: Read file metadata. */
  Fstatat(dirfd, name, sb, AT_SYMLINK_NOFOLLOW);

  print_mode(sb->st_mode);
  printf("%4ld", sb->st_nlink);
  print_uid(sb->st_uid);
  print_gid(sb->st_gid);

  if(S_ISBLK(sb->st_mode) || S_ISCHR(sb->st_mode))

    printf(" %ld", sb->st_size);

  printf("  %s", name);
